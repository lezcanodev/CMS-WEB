Frontend CMS WEB - Grupo 06

Front-End del proyecto Sistema de Administración de Contenido Web (CMS Web).

Tecnologías a utilizar

  • React Testing Library y Vitest se usaran para realizar testing sobre los componentes del GUI y sobre los métodos, clases, etc. (https://testing-library.com/docs/react-testing-library/intro/) (https://vitest.dev/guide/)

  • React-router sera el encargado del enrutamiento de paginas. (https://reactrouter.com/en/main)

  • Redux/toolkit para manejar estados de componente y diferentes lógicas tanto síncronas como asíncronas. (https://redux-toolkit.js.org)

  • Axios para manejar peticiones http, lo cual será útil para la comunicación con la API. (https://axios-http.com/docs/intro)

  • Formik para manejar el estado y control de errores en los formularios. (https://formik.org)

  • Yup para generar los esquemas de validación de datos para cada formulario y trabajara en conjunto con formik. (https://www.npmjs.com/package/yup)

  • Material-ui nos proporciona un conjunto de componentes ya diseñados para utilizar en las diferentes plantillas. (https://mui.com)

  • Typedoc para generar la documentación un HTML que tendrá la documentación del proyecto. (https://typedoc.org/guides/overview/)

Instalación

Sigue los siguientes pasos para instalar el proyecto:

  1. Clonar el repositorio:
    git clone <URL_DEL_REPOSITORIO>
  2. Instalar todas las dependencias: dentro de la carpeta del proyecto ejecutar
    npm install
  3. Correr el servidor de desarrollo: dentro de la carpeta del proyecto ejecutar
    npm run dev

Otros comandos

  • Para generar la documentación ejecutar: esto crea una nueva carpeta llamada docs en la base del proyecto, dentro estara un index.html el cual se tendra que abrir en el navegador.
    npm run generate-doc
